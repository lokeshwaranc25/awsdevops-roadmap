05 · portfolio

38 projects: beginner → production

11 beginner, 11 intermediate, 11 advanced, and 6 production-grade capstones — every single one designed to run inside the AWS Free Tier or for a few dollars of credit, with cost-saving substitutions called out explicitly. Each ships as its own GitHub repo: README → architecture diagram → IaC → CI/CD → screenshots → teardown script.

Free-tier survival rules (apply to every project below)
① Always use t2.micro / t3.micro (750 free hrs/mo, 12 months) — never a bigger instance for labs.
② Skip EKS ($0.10/hr control plane, no free tier) — use k3s or kind on one free EC2 instead.
③ Skip NAT Gateway (~$0.045/hr) — use public subnets or a NAT instance for learning labs.
④ Skip managed OpenSearch (no free tier) — self-host FAISS/Chroma for vector search.
⑤ RDS: single-AZ db.t3.micro only, 20GB — never Multi-AZ in a lab.
⑥ Set a $5 and $20 AWS Budget alert day one, and run terraform destroy / stop instances every night.

09 · interview prep

Phase 13: Interview readiness

Runs in parallel with Phases 11–12 once projects exist to talk about.

Core question categories

  • Linux & scripting: process management, file permissions, cron, sed/awk one-liners
  • AWS scenario design: "design a highly available 3-tier app", "reduce S3 cost by 40%"
  • CI/CD: blue-green vs canary, rollback strategy, pipeline failure debugging
  • Kubernetes: pod scheduling, readiness vs liveness probes, debugging CrashLoopBackOff
  • AI/MLOps: Bedrock vs SageMaker, RAG vs fine-tuning, deploying and rolling back model endpoints
  • IaC: Terraform state locking, drift detection, module design
  • Incident response: "prod is down, walk me through your first 10 minutes"
  • Behavioral (STAR): conflict, ownership, learning from failure
  • System design: design a CI/CD platform, design a logging pipeline at scale

Mock interview cadence

  • Week 1–2 of Phase 13: record yourself answering 20 common questions, review tone & filler words
  • Week 3: 2 peer mock interviews (Pramp / community Discords)
  • Week 4: 1 technical whiteboard-style system design mock
  • Ongoing: keep an "interview log" — every real interview's questions get added back into your prep bank
  • HR round prep: salary research, "why this company", relocation/notice period answers ready
10 · job search

Phase 15: Getting hired

Resume & presence

  • One-page ATS-safe resume: no tables/graphics, standard headers, quantified bullets ("cut deploy time 65%, 12min → 4min")
  • LinkedIn: headline = role + 3 keywords, banner with tech stack, weekly post about a project
  • GitHub: pinned repos = 5 best projects, every repo has a real README with architecture diagram
  • Portfolio site: one page, links to all 6 capstones, deployed on your own AWS account (proof of skill)
  • Apply in batches: 10 tailored applications/day beats 50 generic ones

Outreach strategy

  • Target 3 tiers: startups (faster hiring), mid-size product companies, service/consulting firms
  • Message hiring managers directly with a 3-line note + 1 project link, not a resume dump
  • Contribute to one open-source DevOps tool — visible commits matter more than a cert alone
  • Track every application in the tracker below: status, recruiter, next action date
  • Negotiate: always get 2+ offers in flight before accepting one

12 · sustain it

Habits, AI tools, and mistakes to avoid

Daily & professional habits

  • Commit to GitHub every single day — even a 3-line README update counts
  • Write a one-paragraph "what I learned" log daily — becomes interview material later
  • Read one AWS "What's New" post per day to track service evolution
  • Destroy every lab resource (terraform destroy) before sleeping — cost discipline is a real skill
  • Document incidents like a real engineer: timeline, root cause, fix, prevention

AI tools for productivity

  • Claude / ChatGPT — explain errors, review Terraform/YAML, draft READMEs
  • GitHub Copilot — boilerplate scripts and pipeline YAML scaffolding
  • AWS Q Developer — in-console troubleshooting and IaC suggestions
  • Amazon Bedrock / SageMaker — the AI services you'll deploy, not just use as a coding aid
  • Notion AI / Obsidian — structure your daily learning log automatically

Mistakes to avoid

  • Tutorial-hopping without finishing a single project end-to-end
  • Skipping Linux/networking fundamentals to "rush" to AWS
  • Leaving AWS resources running — surprise bills kill motivation
  • Collecting certs with zero hands-on labs behind them
  • Copy-pasting Terraform/Jenkinsfiles without understanding every line
  • Ignoring cost optimization and security until "later"
  • Studying 10 hours one day, zero the next 3 — consistency beats intensity
  • Applying to jobs only after feeling "100% ready" — start at 70%
